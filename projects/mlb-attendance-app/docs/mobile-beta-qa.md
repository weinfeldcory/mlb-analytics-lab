# Mobile Beta QA

Widths to review:
- 320px
- 375px
- 390px
- 414px
- 430px

Global checks:
- No horizontal overflow on first load.
- Primary CTA is visible without hunting.
- Tap targets are comfortable.
- Keyboard does not cover the active input or save action.
- Error and empty states are readable.

Auth:
- Hero copy wraps cleanly.
- Display-name and password inputs remain fully visible with keyboard open.
- Create-account username preview does not clip.
- Log in / Create Account button stays reachable.

Onboarding:
- Step progress dots stay on screen.
- Generated `@username` preview wraps cleanly.
- Favorite-team cards stack without clipping.
- Final `Log your first game` CTA is visible and tappable.

Home:
- Hero metrics stay within card bounds.
- Primary CTA remains obvious above the fold.
- Insight/prompt cards do not overflow horizontally.

Log Game:
- Search chips wrap cleanly.
- Search inputs stack cleanly at phone widths.
- Result cards remain readable and selectable.
- Seat-detail chips wrap cleanly.
- Quick memory sparks are fully visible.
- Sticky `Save Game` CTA stays visible and tappable.

Log Recap:
- Hero summary fits without clipping.
- Stat cards stack vertically on phones.
- Next-action buttons are visible and easy to tap.

History:
- Filter controls stack cleanly.
- Moment cards remain readable.
- No dense inning-by-inning table is forced above the fold on phone widths.
- Edit and delete actions remain reachable.

Following:
- Search input and result cards fit comfortably.
- Follow / Following buttons do not squeeze text off screen.
- Activity cards read cleanly in one column.
- Following and Followers lists stay card-based.

Fan Résumé:
- Top summary reads cleanly without horizontal scrolling.
- Any dense detail blocks are below the fold or collapsed enough to scan.

Profile:
- Identity card, generated username preview, and buttons fit without clipping.
- Sign out remains visible and tappable.
- Following preview cards stack cleanly.
